# Design ‚Äî Workflow IR build path (tau-ui)

**Date:** 2026-06-13
**Repo:** tau-ui (`gateway/` + `web/`)
**Companion spec:** `2026-06-13-tau-ir-inspect-verb-spec.md` (the tau PR)
**Track:** handoff `03-workflow-ir-build-path.md`

Light up the GraphEditor's gated controls so the UI can (a) show the **compiled
project IR** (resolved tools, lowered subflow edges, capability table, target) ‚Äî
not just the source TOML ‚Äî and (b) prove a workflow compiles reproducibly via an
interim build slice that ships **before** the tau verb lands.

## 1. Key constraint that shaped this design

tau's IR is **project-scoped**: `lower_project` compiles the whole `tau.toml`
into one `IrModule`. `workflows/<name>.toml` files are a *separate orchestration
model* (the `tau-workflow` crate), which is what the GraphEditor draws today.
They are different graphs. Therefore:

- The compiled view is the **project** IR, not a per-workflow graph.
- We bridge the scope gap in the **web**: the Compiled view highlights the agents
  the *currently selected workflow* references (known from the Source graph),
  dims the rest. Zero added complexity in tau or the gateway.
- The verb is project-scoped (`tau ir inspect`, no `--workflow`); the gateway IR
  route is project-scoped (`GET /api/projects/{pid}/ir`).

## 2. The real delta (what Compiled adds over Source)

Repo-true, from the `IrModule` fields that actually exist:
1. **Capability table** ‚Äî per-tool capability requirements (chips on tool nodes).
2. **Lowered subflow edges** ‚Äî `edges: Vec<SubflowEdge>` (agent‚Üítool wiring).
3. **Resolved per-agent tool set** ‚Äî `tools` + each agent's references.
4. **Target triple** the project lowered for.

`pruned_packages` was speculative and is **not** an `IrModule` field ‚Äî dropped.

## 3. Architecture (follows existing seam pattern)

```
web GraphEditor ‚îÄ‚îÄHTTP‚îÄ‚îÄ> gateway /api/projects/{pid}/ir ‚îÄ‚îÄ> AppState.compiled_ir()
                                                              ‚îî‚îÄ> IrSource (Mock | Cli)
                                                                    Cli: shell `tau ir inspect --json`, parse envelope.module
                                                                    Mock: return checked-in fixture IrModule

web GraphEditor "Build" ‚îÄ‚îÄHTTP‚îÄ‚îÄ> existing /build (ShipSource) ‚îÄ‚îÄ interim slice, no new gateway code
```

### 3.1 Gateway ‚Äî new `ir` seam (`gateway/src/ir/mod.rs`)

Mirrors `WorkflowGraphSource`/`ShipSource`:

```rust
pub trait IrSource: Send + Sync {
    fn inspect(&self) -> Result<CompiledIr, IrError>;   // project-scoped, no args
}
pub struct MockIr;                                       // returns the fixture below
pub struct CliIr { bin: PathBuf, project: PathBuf }      // shells `tau ir inspect --json`
```

- **`CompiledIr`** ‚Äî a gateway-owned, `ts-rs`-exported projection of the verb's
  JSON (the gateway does **not** link tau-ir; it defines its own
  `#[derive(Deserialize)]` structs matching the envelope's `module`). Fields the
  UI renders only:
  ```rust
  pub struct CompiledIr {
      pub hash_kind: String,           // "lowered" | "bundle"
      pub canonical_ir_hash: String,
      pub target: String,
      pub tau_version: String,
      pub agents: Vec<IrAgent>,        // { id, llm_backend, tools: Vec<String> }
      pub tools: Vec<IrTool>,          // { id, capabilities: Vec<String> }
      pub edges: Vec<IrEdge>,          // { from, to, kind }
  }
  ```
  `CliIr::inspect` runs `Command::new(bin).args(["ir","inspect","--json"]).current_dir(project)`,
  parses the envelope, and **projects** `module` ‚Üí `CompiledIr` (flatten the
  BTreeMaps to vecs, fold `capability_table` into per-tool `capabilities`, derive
  each agent's `tools` from its references). This projection is the only "logic"
  in the seam ‚Äî deterministic, unit-tested against a fixture.
- **`IrError`** (thiserror): `NotCompiled`/`Lower(String)` (verb exit 2 / stderr),
  `Spawn(String)` (couldn't run tau), `Parse(String)` (bad JSON). Mapped in the
  handler: `Lower`‚Üí422, `NotCompiled`‚Üí422, others‚Üí500.
- **`MockIr`** returns a hand-authored fixture (see ¬ß5) so the entire feature is
  buildable/testable before the verb exists.

### 3.2 Gateway ‚Äî wiring

- `state.rs`: add `ir_source: Box<dyn IrSource>` to `Inner`; in `with_options`,
  `let ir_source = if is_mock { Box::new(ir::MockIr) } else {
  Box::new(ir::CliIr::new(bin.clone(), project.clone())) };`. Add
  `AppState::compiled_ir(&self) -> Result<CompiledIr, IrError>` delegating to it.
- `api/ir.rs`: `pub async fn compiled(Scoped(state)) -> Result<Json<CompiledIr>,
  (StatusCode,String)>`.
- `api/mod.rs`: `pub mod ir;` + `.route("/ir", get(ir::compiled))` in the scoped
  router.

### 3.3 Web

- `web/src/api/ir.ts`: `export const getCompiledIr = (pid) =>
  request<CompiledIr>(scopedPath(pid, "/ir"))`.
- `web/src/types/CompiledIr.ts` (+ `IrAgent`/`IrTool`/`IrEdge`) generated by
  `ts-rs`.
- **GraphEditor toggle:** a `Source ‚Üî Compiled IR` segmented control above the
  canvas.
  - **Source** (default): existing `getWorkflowGraph` + `workflowToFlow`,
    editable (unchanged behaviour).
  - **Compiled IR**: fetch `getCompiledIr(pid)`; lay agents/tools into flow nodes
    (new `irToFlow` in `graph/layout.ts`), tool nodes carry capability chips,
    subflow edges styled distinctly; **read-only**. Agents whose id ‚àà the current
    workflow's agent set (collected from the Source graph's `agent.run` nodes) are
    emphasized; others dimmed. Header shows `target` + `hash_kind`/short hash.
  - Reuse `GraphCanvas`; add an IR node renderer (or a `variant` prop) for chips.
- **Buttons:**
  - Replace the disabled `üîí Build from IR` with a real **Build** button calling
    the **existing** ship `build(pid, target)` API; show `shortHash(sha256)` +
    target on success (reuse `ShipPage` helpers/patterns). Ships in Phase 0.
  - Remove the disabled `üîí Save ‚Üí IR` button. Keep the local-edit banner but drop
    the "Save ‚Üí IR waits on tau Œ≤.2" text (Save/graph‚ÜíTOML write is out of scope ‚Äî
    see ¬ß6).
- **Sidebar:** remove `gated: true` from the `workflows` item **only in Phase 2**
  (when the real Cli IR path is the default). Phase 0/1 leave it gated.

## 4. Sequencing ‚Äî what's blocked vs not

The Mock seam + fixtures make most work **non-blocked**. Only the switch to the
real Cli data path and the final un-gate require the merged verb.

```
PHASE 0  (now, no verb dependency)            PARALLEL with the tau PR
  P0-tau   implement `tau ir inspect`  ........  separate tau PR  ‚Üê THE GATE
  P0-ui    GraphEditor interim "Build" button ‚Üí existing /build; remove Save‚ÜíIR
           disabled button; banner copy. Tests. (prettier gate.)  SHIPPABLE NOW.

PHASE 1  (build against Mock/fixture ‚Äî also not blocked on the verb)
  P1-gw    gateway `ir` seam: IrSource + MockIr + CliIr(projection) + /ir route +
           state wiring + ts-rs types. Unit-test CliIr projection against the
           checked-in fixture; API integration against MockIr. (No real binary.)
  P1-web   GraphEditor Source‚ÜîCompiled toggle + irToFlow + chips + highlight,
           against MockIr. vitest with stubbed fetch. (prettier gate.)

  ‚îÄ‚îÄ tau PR MERGES here ‚îÄ‚îÄ  (the only true gate)

PHASE 2  (requires the merged verb)
  P2-verify  no code flip needed ‚Äî `is_mock` is global/autodetected, so real
             mode uses `CliIr` automatically once the verb exists. Add the
             TAU_REAL_BIN-gated end-to-end test exercising `tau ir inspect`, and
             reconcile the fixture with the verb's real serde shape if it drifted.
  P2-gate    remove `gated` badge (Sidebar) + any remaining disabled affordance,
             now that the real compiled render works against real tau.
```

Dependencies: P1-web ‚Üí P1-gw (route+types). P2-verify ‚Üí P0-tau (merged) + P1-gw +
P1-web. P2-gate ‚Üí P2-verify. P0-ui is independent and ships first.

## 6. Out of scope (noted, not built)

- **Save ‚Üí IR (graph‚ÜíTOML write):** lossy round-trip from the simplified editor
  graph + a new gateway write endpoint = a separate authoring track. Editing
  stays local for now.
- **`tau workflow resume <RUN_ID>` / `tau workflow log`:** separate gap (handoff).
- **Cross-target conformance (HealthPage, "waits on Œ≤.6"):** not shipped ‚Äî leave
  gated.
- **Per-workflow IR filtering in tau** (`--agent`): handled by web highlighting
  instead.

## 7. Test strategy summary

| Layer | Test | Backed by |
|---|---|---|
| gateway | `CliIr` projection (envelope‚Üí`CompiledIr`) | checked-in fixture `ir-inspect.json` |
| gateway | `/ir` route returns agents/tools/edges/caps | `MockIr` (`is_mock=true`) |
| gateway | real verb end-to-end | `TAU_REAL_BIN`-gated, skipped in CI |
| web | Compiled tab renders chips + subflow edges + highlight; Source‚ÜîCompiled toggle; Build shows hash | vitest, stubbed `fetch` |

Every frontend task runs `prettier` in its gate (per-task gates omit
`format:check`).
